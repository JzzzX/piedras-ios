# Piedras Cloud 首页与后台精简设计

日期：2026-03-27

## 目标

把 `https://piedras.preview.aliyun-zeabur.cn/` 从“状态页 + 全量后台管理混在一起”的旧页面，改成和当前产品阶段匹配的简洁入口。

本轮目标：

- 首页 `/` 只展示产品云端入口与运行状态
- 真正的后台管理迁移到独立的 `/admin`
- `/admin` 只保留当前账号系统必需的最小后台
- 去掉邀请码、legacy 数据接管、托管账号创建等当前不用的管理能力
- 保持现有服务状态读取与管理员登录能力可用

## 非目标

- 本轮不重做后端认证接口
- 本轮不增加新的后台功能
- 本轮不做完整对外官网
- 本轮不恢复邀请码相关入口
- 本轮不处理 legacy 数据迁移工作流

## 信息架构

### 1. 首页 `/`

首页是一个非常简洁的云端入口页，不再承载后台表单和管理表格。

保留内容：

- 产品标识：`Piedras Cloud`
- 一句说明：这里是 iOS 产品依赖的云端入口与状态页
- 三个状态卡片：
  - `Database`
  - `LLM`
  - `ASR`
- 少量常用入口：
  - `/healthz`
  - `/api/asr/status`
  - `/api/llm/status`
- 一个清晰的“进入后台”按钮，跳转 `/admin`

移除内容：

- 管理员登录框
- 账号列表
- 邀请码管理
- legacy 工作区管理
- 账号创建和密码重置表单

### 2. 后台 `/admin`

`/admin` 变成真正独立的后台页面，不再重定向回首页锚点。

未登录时：

- 显示管理员登录框

已登录后：

- 顶部显示后台标题与退出按钮
- 显示最小系统摘要：
  - schema 状态
  - 账号数量
  - Database / LLM / ASR 运行状态
- 显示账号列表
- 每个账号保留“重置密码”操作

从后台删除：

- 邀请码生成
- 邀请码列表
- legacy 工作区现状
- legacy 工作区接管
- 托管账号创建
- 大段部署命令说明

## 视觉方向

首页采用用户已确认的 `A1 极简状态页` 风格。

要求：

- 视觉安静，少噪音
- 不使用当前偏重的“玻璃后台”堆叠感
- 首页更像产品云入口，而不是内部运维面板
- 后台也保持简洁，但仍然允许清楚扫描账号信息

建议方向：

- 首页使用浅暖底色 + 稳定排版
- 后台继续沿用当前深色体系，但层级减少、区块更明确

## 技术方案

### 路由调整

- `app/page.tsx` 改为只渲染首页状态入口
- `app/admin/page.tsx` 不再重定向，改为直接渲染独立后台页

### 组件调整

- 复用现有状态读取逻辑：数据库、ASR、LLM、管理员 session
- `AdminConsole` 精简为最小后台组件
- 如有必要，新增一个首页专用组件，避免首页和后台继续耦合

### 数据读取

首页仍然读取：

- 数据库连通性
- ASR 状态
- LLM provider 状态

后台仍然读取：

- 管理员 session
- 账号 dashboard 数据

但后台界面只消费最小必要字段。

## 验收标准

满足以下条件才算完成：

- 访问 `/` 时不再看到后台管理内容
- 首页可以清楚看到产品标识、状态卡片和后台入口
- 访问 `/admin` 时可以独立看到管理员登录页
- 管理员登录后能看到账号列表和密码重置入口
- 页面中不再出现邀请码和 legacy 接管相关 UI
- 现有健康检查与状态读取不回退

## 风险

- `AdminConsole` 当前承载内容较多，精简时要避免误删登录和密码重置能力
- `/admin` 目前是重定向页，改成真实页面后要确认 query/message/error 参数仍可正确显示
- 首页与后台共用样式时，需避免把首页 снова改成“后台感”页面
